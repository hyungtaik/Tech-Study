# 객체지향 SW 흐름

1. 초창기 SW는 HW를 실행하기 위한 기능 중심의 프로그램에 집중하게 되어 **구조적 프로그램 기법**이 발달하게 된다.

   - C언어의 발달

   

2. 다양한 분야에 컴퓨터가 사용되어지기 시작하면서 SW란 단순히 기능 단위의 처리 중심이 아닌 데이터를 입력받아 처리하고, 그 결과 데이터를 다시 만들어내는 것에 착안하여, **프로그래밍의 중심이 Data에 있다**는 **정보 공학 프로그램 기법**이 발달하면서 DBMS가 활성화되었다.

   

3. 1,2,와 같은 방법으로 작성된 프로그램들은 데이터와 기능이 따로 관리되어 복잡한 관계성을 갖기 때문에 나중에 유지보수성이나, 확장성, 재사용성에 좋지 못한 결과를 가져오게 되었으며, 좀 더 품질 좋은 SW를 만들수 있는 **객체지향 기법**을 만들게 된다. 

   - C++, Java 발달

   

4. 객체지향을 적용한 개발은 재사용성의 증가를 가져와 개발비 절감과 품질 보증에 이점을 가져온다. 그러나 더 많은 재사용성의 효과를 가져오기 위해 재사용 단위가 큰 **Component 기반의 개발 기법**이 등장하게 된다. (CBD발달) 

   * CBD - Componet Based Development 의 약자로, 재사용 가능한 컴포넌트의 개발 또는 사용 컴포넌트들을 조합하여 애플리케이션 개발 생산성, 품질을 높이고, 시스템 유지보수 비용을 최소화 할 수있는 혁신 개발방법론

   * 컴포넌트 - 특정 기능을 수행하는 모듈로서 한 시스템에 종속적이지 않아 재사용 가능하며 교체가 가능해야한다. 또한 컴포넌트는 여러개의 함수, 여러개의 클래스로 구현되며 인터페이스를 통해서만 접근하도록 하여 쉽게 교체할 수 있도록 한다.

     

5. 재사용성에 대한 꾸준한 연구로 견고한 프로젝트와 품질좋은 프로젝트를 빠르게 개발하기 위해 **입증된 Pattern을 적용하여 SW 아키텍쳐를 구현**하고, **필요한 라이브러리(API)를 추가한 Framework**가 만들어지게 되어 재사용되게 된다.

   - Framework 기반의 개발은 검증된 안정된 기반에서 애플리케이션들이 작동되도록 환경을 제공한다.



---



* 모든 제품의 개발은 비용 절감과 품질 향상을 위해 노력하게 된다. 그러나 비용 절감을 하다 보면 품질에 문제가 생길 수 있고, 품질 향상을 신경쓰다보면 많은 비용이 필요하게 된다. 이러한 두 조건을 만족시키기 위한 좋은 해결책은 **재사용(Reuse)**에 있다. 

  재사용을 하게 되면 조금 **더 빠르게 제품을 생산**할 수 있고, 좋은 품질의 코드만을 재사용하므로 **품질 향상 효과**를 같이 가져오게된다.

* 객체지향 개념은 개발자에게 **쉽게 커스터마이징이 가능**하도록 하고, **유지보수성, 확장성, 재사용성을 증가** 준다.



---

### 📎 용어 정리

> - Framework: 기본적인 SW의 구조를 구현하고, 프로그래밍에 필요한 라이브러리를 포함하고 있는 반제품 형태의 SW를 의미한다. 잘 짜인 구조가 미리 프로그래밍화 되어 있기 때문에 안정적이고, 빠른 개발을 할 수 있게 한다. 
> - Application Framework: 프로그래밍에서 특정 OS를 위한 응용 프로그램 표준 구조를 구현하는 클래스와 라이브러리 모임이다.
> - Platform: 컴퓨팅에서 플랫폼은 소프트웨어가 구동 가능한 하드웨어 아키텍처나 소프트웨어 프레임워크의 종류를 설명하는 단어이다.  일반적으로 컴퓨터의 아키텍처, OS, 프로그램 언어, 관련 런타임 라이브러리 또는 GUI를 포함한다.
